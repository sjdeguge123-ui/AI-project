# 全文文案模式 · 跨格式视觉规范（Design Spec）

> 适用范围：VidGrab「全文文案（fulltext）」模式的五种导出格式 —— Markdown(.md) / HTML(.html) / Word(.docx) / PDF(.pdf) / 图片(.png)
> 目标：五种格式在「时间戳样式、片段分隔、段间距、中文标点」上视觉一致、可读性强。
> 本文为**设计规范**，不含实现代码；工程师据第 7 节映射表逐格式落地。

---

## 1. 问题诊断（为什么现在不一致）

全文文案的源数据由 `core/summarizer.py:_build_full_text()` 生成：每个时间戳片段之间仅用**单个 `\n`** 分隔，片段内部无换行。各渲染器对单个 `\n` 的处理不同，导致五格式表现分裂：

| 格式 | 当前渲染逻辑 | 结果 |
|------|--------------|------|
| HTML | `md.markdown()` 把单 `\n` 折叠成空格 | ❌ 所有文字连成一片，时间戳与正文无换行（用户反馈的核心 bug） |
| Markdown 文件 | 原样写入单 `\n`；任何 Markdown 渲染器同样折叠 | ❌ 与 HTML 同样连成一片（仅纯文本编辑器里看得见换行） |
| PDF | `full_text.split("\n")` 逐行成 Paragraph | ✅ 有分段，但分段粒度依赖单 `\n` |
| Word | `split("\n")` 逐行成段落 | ✅ 有分段 |
| PNG | `split("\n")` 逐行绘制 | ✅ 有分段 |

**根因**：① 分隔符用「单 `\n`」而非「段落/空行」，Markdown/HTML 语义下不生效；② 五种渲染器各自实现分段逻辑，没有统一的数据模型与统一的切分函数。

**解决思路（第 6 节展开）**：定义统一的「时间戳片段」数据模型 + 一个被五种渲染器共享的切分函数；片段间用**段落级分隔**（空行 / 独立 `<p>` / 独立段落 / 固定垂直间距）渲染。

---

## 2. 统一数据模型：时间戳片段（Timed Segment）

将全文文案规范为**有序的片段序列**，每个片段结构如下：

```
TimedSegment = {
    timestamp: "[MM:SS]"        # 必填，章节/自然停顿处的时间标记
    text:      "该片段正文……"   # 必填，带完整中文标点的连续文字
}
```

- 一个片段 = 一个时间戳 + 其后的正文，直到下一个时间戳之前。
- 片段之间**天然分段**（不需要用户手动分段）。
- 原始 `full_text` 字符串在内部应可无损地切分为该序列；推荐导出层统一调用共享切分函数（见 6.3）。

---

## 3. 时间戳 `[MM:SS]` 视觉规范

| 属性 | 规范值 | 说明 |
|------|--------|------|
| 格式 | `[MM:SS]`（或 `[HH:MM:SS]`） | 与现有 `_TS_RE` 正则一致 |
| 颜色 | `#E74C3C`（关键词红） | **复用现有 strong / 关键词红色**，保证品牌一致 |
| 字重 | 粗体（bold） | 与关键词高亮一致 |
| 字号 | 与正文相同 | 不放大、不缩小，保持行内连贯 |
| 位置 | **行内段首**，不独占一行 | 扫读时一眼定位时间，又不打断阅读流 |
| 与正文间距 | 时间戳后跟 **1 个半角空格（U+0020）** | 如 `[02:30] 很多人问我……`；禁止在 `]` 后加全角空格或多余间距 |
| 标点 | 时间戳本身不算句读，后方正文以正常中文标点起始 | 不因为时间戳而在其后加逗号/句号 |

**可选增强（非必需）**：HTML / Markdown 中把时间戳做成可点击深链，跳转到视频对应时刻（如 B 站 `?t=<秒>`）。实现与否不影响本规范一致性，建议后续作为增强项。

---

## 4. 片段分隔与段间距

每个 TimedSegment 渲染为**一个独立段落/区块**，片段之间用段落级间距，而非行内换行。

| 格式 | 片段间分隔方式 | 段间距（推荐） |
|------|----------------|----------------|
| Markdown | 片段间空一行（`\n\n`） | 由渲染器默认段落间距 |
| HTML | 每个片段一个 `<p>`，时间戳 span 在 `<p>` 内 | `margin-bottom: 0.6em`（约 9–11px @16px 字号） |
| Word | 每个片段一个独立 paragraph | `space_after = 6 pt` |
| PDF | 每个片段一个独立 Paragraph | `spaceAfter = 6`（reportlab 单位，约 6pt） |
| PNG | 片段间固定垂直留白 | `8–10 px`（绘制时 `y += h + 8`） |

**正文行内**：片段内部若因超宽折行，按各格式正常 CJK 折行处理，**不**插入额外空行。

---

## 5. 中文标点与排版规范

全文文案为「完整转录原文」，可读性依赖标点。规范如下：

### 5.1 标点完整性（内容层要求）
- 全文文案**必须带完整中文标点**。来源优先级：
  1. 转录文本本身已带标点（如 B 站字幕 JSON3、多数平台字幕含标点）→ 直接采用；
  2. ASR 原文无标点时，须在转录阶段或导出前补一次**标点恢复**（方案：轻量 LLM 单call / 本地标点恢复模型 / 规则兜底）。因 fulltext 模式为省 Token 跳过 AI 摘要，标点恢复可做成可选项，但**默认应保证有标点**（用户已明确反馈缺标点影响阅读）。
- 每个片段结尾应有句末标点（`。？！…`）；片段内部用 `，` `；` 做自然分句。
- 长片段（>80–120 字）建议在下一个时间戳处自然断段（已由时间戳天然分段覆盖）。

### 5.2 中文排版细则（渲染层要求）
- **字间不加空格**：CJK 字符之间不插入任何空格。
- **中文标点用全角**：`。，；：？！""''（）` 等用全角；不混用半角 `, . ; :`。
- **标点后不加空格**：`。，；：` 等后不跟空格；`"` 内正常排布。
- **中英文/数字混排**：中文与 ASCII 字母、数字之间可按习惯加半角空格（如 `Blender 软件`、`3 分钟`），保持与现有正文一致即可，不强制。
- **行高**：正文 `line-height` 1.6–1.8（HTML 1.8、PDF leading 16/10pt≈1.6、PNG 1.4–1.5×、Word 1.5×）。

## 6. 一致性落地原则（给工程师的约束）

### 6.1 单一事实源 + 共享切分
- 定义**一个** `split_full_text_segments(full_text) -> List[TimedSegment]`，被 HTML / Word / PDF / PNG / Markdown 五处渲染**共同调用**，保证分段逻辑唯一。
- `full_text` 内部存储可用「片段间双换行 `\n\n`」或「片段间单 `\n` 但切分时按时间戳正则 `_TS_RE` 切」——但**渲染层一律以 `_TS_RE` 命中处为片段起点**，不依赖 `\n` 数量。这样无论源数据用单/双换行都能正确分段。

### 6.2 HTML 修复要点（关键）
- 不要直接把含单 `\n` 的 full_text 丢给 `md.markdown()`（会被折叠）。
- 改为：先用 `split_full_text_segments` 切成片段 → 每个片段生成 `<p><span class="ts">[MM:SS]</span> 正文</span></p>` → 拼入 HTML body。
- 在 `<style>` 中新增 `.ts { color:#E74C3C; font-weight:bold; }` 与 `p { margin: 0 0 0.6em; }`，与现有 `strong{color:#e74c3c}` 风格统一。

### 6.3 Markdown 修复要点
- full_text 写入 `.md` 时，片段间用**空行（`\n\n`）**分隔，使任何 Markdown 渲染器都呈现为段落；时间戳用 `**[MM:SS]**`（粗体；Markdown 原生不支持字体色，粗体已足够与正文区分，且各渲染器通用）。

### 6.4 颜色 / 字体令牌（与现有设计对齐）
- 时间戳红 = 关键词红 = `#E74C3C`
- 正文色 = `#333`（HTML）/ `#2C3E50`（标题）/ PDF `STSong-Light` 黑字
- 强调蓝（标题下划线等）= `#4A90D9`
- 字体族（HTML/PNG）：`-apple-system, "Microsoft YaHei", "PingFang SC", sans-serif`；PDF 用 `STSong-Light`；Word 用系统默认中文字体（等线/宋体）。

## 7. 五格式一致性映射表（验收基准）

| 维度 | Markdown | HTML | Word(.docx) | PDF | PNG |
|------|----------|------|-------------|-----|-----|
| 片段分隔 | 空行 `\n\n` | 独立 `<p>` | 独立 paragraph | 独立 Paragraph | 独立绘制块 |
| 段间距 | 渲染器默认 | `0.6em` | `6pt` | `6` | `8–10px` |
| 时间戳颜色 | （粗体，无色） | `#E74C3C` | `#E74C3C` | `#E74C3C` | `#E74C3C` |
| 时间戳字重 | 粗体 `**` | bold | bold | bold | bold |
| 时间戳-正文间距 | 1 空格 | 1 空格 | 1 空格 | 1 空格 | 1 空格 |
| 正文标点 | 全角中文标点 | 同左 | 同左 | 同左 | 同左 |
| 字间空格 | 无 | 无 | 无 | 无 | 无 |
| 行高 | 1.8 | 1.8 | 1.5× | 1.6 | 1.4–1.5× |
| 字号(正文) | 16–17px | 16–17px | 11pt | 10pt | 17px |

> 说明：Markdown 原生不支持字体颜色，故时间戳仅以粗体区分（彩色在渲染器支持时可选）；其余四格式均显红，达到「视觉一致」诉求。

---

## 8. 自检清单（验收用）

- [ ] HTML 中时间戳与正文、**不同片段之间**均有清晰换行/段落，不再连成一片。
- [ ] 五格式每一个时间戳片段都是独立段落/区块，段间距符合上表。
- [ ] 时间戳统一为红+粗，位于片段段首、后接 1 空格。
- [ ] 全文文案含完整中文标点（。，；：？！），无字间多余空格。
- [ ] 中英文/数字混排符合既有正文习惯。
- [ ] 五格式并排对比，时间戳样式、分段、段间距、标点观感一致。
- [ ] 共享切分函数 `split_full_text_segments` 被五格式复用，无各自为政的分段逻辑。

---

## 9. 与产品/其他角色的接口约定

- **内容层（summarizer）**：保证 `full_text` 含标点；时间戳位置合理（每隔约 3–5 分钟或自然停顿）。切分以 `_TS_RE` 为唯一依据，渲染层不依赖换行数量。
- **本规范不与「精简/详细」模式的表格呈现冲突**：仅约束 fulltext 模式的全文文案区块。
- **金句模块**：金句列表中的 `[MM:SS]` 同样套用上述时间戳红+粗样式，保持一致。
