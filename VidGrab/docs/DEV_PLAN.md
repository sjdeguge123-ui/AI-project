## 二、各阶段详细计划

### Phase 0 — Skill MVP（第 1-2 周）
**目标**：打通核心链路，能用，跑通 YouTube + B站 → Markdown 输出

| 序号 | 任务 | 说明 |
|---|---|---|
| 0-1 | 初始化 Git 仓库，搭建目录结构 | GitHub + Gitee 双库 |
| 0-2 | 链接平台识别模块 | 判断是B站/YouTube/其他 |
| 0-3 | YouTube 字幕提取 | yt-dlp，优先有字幕 |
| 0-4 | B站字幕提取 | bilibili-api |
| 0-5 | 无字幕视频音频提取 + Whisper 转录 | 支持 API版 和 本地版 |
| 0-6 | LiteLLM 接入，AI摘要生成 | 先接 DeepSeek 验证流程 |
| 0-7 | 结构化输出 Markdown（含时间戳） | 按定稿模板输出 |
| 0-8 | 封装为 CodeBuddy Skill | skill.json + 入口 |

**关键节点**：能在 CodeBuddy 里粘贴链接，拿到 Markdown 摘要。

**需要你做的事**：
- 确认 GitHub 仓库已创建并可推送
- 准备 DeepSeek API Key（测试用）
- 提供 2-3 个测试链接（B站有字幕、YouTube有字幕、无字幕视频各一个）

---

### Phase 1 — Skill 完整版（第 3-4 周）
**目标**：补全所有输出格式，完善用户体验，发布到 CodeBuddy Skill 市场

| 序号 | 任务 | 说明 |
|---|---|---|
| 1-1 | PDF 导出 | WeasyPrint |
| 1-2 | Word 导出 | python-docx |
| 1-3 | 图片导出（简洁文字卡片，白底黑字） | Pillow |
| 1-4 | 多模型接入（GPT/Claude/Gemini/Qwen） | LiteLLM 统一接口 |
| 1-5 | Key 配置引导界面（首次使用弹出） | 图文引导，推荐 DeepSeek |
| 1-6 | 输出语言选择（中文/英文/跟随视频） | Prompt 控制 |
| 1-7 | 长视频分段摘要（超过1小时） | 分段→合并策略 |
| 1-8 | 一键安装脚本（Windows .bat / Mac .sh） | 自动配置 MCP Server |
| 1-9 | MCP Server 封装 | 兼容 Claude/Cursor/Windsurf/Kimi |
| 1-10 | README + 使用文档（中英双语） | 含 Key 申请图文教程 |
| 1-11 | 提交 CodeBuddy Skill 市场审核 | — |

**需要你做的事**：
- 审核 README 和使用文档
- 测试图片卡片样式
- 在自己电脑上跑一遍安装脚本
- 提交市场时需要 CodeBuddy 开发者账号
